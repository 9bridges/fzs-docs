# 典型应用场景

FZS 数据同步平台凭借其灵活的架构和强大的数据处理能力，广泛应用于以下核心业务场景。

## 1. 数据异构集成与汇总

在大型企业或政府机构中，业务系统往往分散在不同的部门和地域，使用着不同类型的数据库（如 Oracle, MySQL, SQL Server 等）。FZS 能够：

- **打破数据孤岛**：实时捕获各业务系统的变更数据。
- **异构转换**：自动处理不同数据库之间的数据类型映射与转换。
- **数据汇聚**：将多源异构数据实时汇聚到统一的数据仓库或大数据平台（如 Hadoop, Kafka），为 BI 分析和决策提供实时数据支撑。

## 2. 核心业务同构/异构容灾

数据是企业的核心资产，业务连续性至关重要。FZS 提供可靠的灾备解决方案：

- **实时热备**：实现生产中心到灾备中心的毫秒级数据同步。
- **双活/多活支持**：支持双向同步，助力构建双活或多地多活的高可用架构。
- **应急切换**：当主库发生故障时，可快速切换至备库，保障业务不中断 (RPO ≈ 0, RTO 极低)。

## 3. 信创数据库平滑迁移

随着信创产业的发展，越来越多的企业需要将核心业务从传统商业数据库迁移至国产数据库（如达梦, 某人大金仓, OpenGauss, TiDB 等）。FZS 在此过程中扮演关键角色：

- **全量+增量无缝迁移**：在不停止业务的情况下，先进行历史全量数据迁移，同时捕获并通过增量数据追平差异。
- **业务割接保障**：提供数据校验功能，确保新旧库数据一致，支持随时回退，极大降低割接风险。
- **国产化适配**：深度适配数十种国产数据库，解决复杂的兼容性问题。

## 4. 实时数据分发与共享

对于需要将核心交易数据实时分发给查询系统、报表系统或第三方合作伙伴的场景：

- **读写分离**：将查询压力分流到从库，保护主库性能。
- **数据脱敏**：在同步过程中对敏感信息（如身份证、手机号）进行实时脱敏，确保数据共享过程中的合规与安全。
