# 数据节点管理

**数据节点 (Data Node)** 是对数据库连接的抽象。在配置数据链路之前，您必须先将源数据库和目标数据库注册为数据节点。

FZS Web 提供了统一的界面来管理各类异构数据库（如 Oracle, MySQL, SQL Server, 达梦等）的连接信息。

## 节点管理
* **[添加节点](./add.md)**：注册新的数据库连接。
* **[编辑节点](./edit.md)**：修改数据库连接信息（如账号、密码、地址）。
* **[删除节点](./delete.md)**：移除废弃的数据库连接。

## 连通性检查
确保 FZS 系统能够正常访问数据库。

* **[Web Server 连通性](./ping-webserver.md)**：检查 FZS Web Server 是否能连接到 Agent。
* **[数据库连通性](./ping-db.md)**：检查 FZS Agent 是否能连接到实际的数据库实例。

:::tip
建议在创建数据链路前，先对相关节点进行连通性检查，以排查网络或权限问题。
:::
